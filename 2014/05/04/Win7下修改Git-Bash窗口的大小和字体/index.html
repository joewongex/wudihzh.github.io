Win7下修改Git Bash窗口的大小和字体

操作系统：Windows7 旗舰版 x64

Git版本：1.8.5.2-preview20131230

刚安装好Git，打开Git Bash的命令行窗口，发现无论是窗体的大小还是字体大小，Git Bash的默认配置都显得太不考虑我们这些高度近视码农的感受了，这么小的字看着实在是伤眼，而且输出缓冲区也不够大，如果项目文件很多，Git命令的输出信息就无法通过滚屏全部查看到，因此，安装完Git后，总得进行一番设置。

按照使用CMD命令行窗口的习惯，一般是通过点击窗口左上角的小图标，在弹出的菜单中选择属性子菜单对窗口的大小、字体及窗口的颜色进行设置，就像这样：

按照同样的套路，对Git Bash命令窗口进行设置，设置完成后，点“确定”按钮后，却收到了这样的错误提示信息：

一头雾水，一阵Google，同样找不到任何答案，偶然之间发现藏在开始菜单中的Git Bash图标，右键单击它弹出的菜单中也有属性选项，就像这样：

打开Git Bash属性配置窗口后，发现这里可以修改的选项比上面提到的方法的多得多，不仅可以修改窗口的大小，修改字体、颜色，还可以改变内容的编码，具体内容大家切换下不同的标签就可以看到了。

按照自己的喜好修改好后，点”应用“、”确定“就可以生效了，最终效果如下图，是不是看起来舒服多了？
